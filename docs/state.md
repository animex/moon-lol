移动系统不关心是谁下达的移动指令，只关心移动路径和移动速度，负责按帧修改 transform 组件。
攻击系统不关心与目标相隔多远，只关心攻击谁，负责攻击状态（前摇、冷却）的更新。
寻路系统负责找到与目标之间的路径，找到了就命令移动系统按路径进行移动。
自动攻击系统通过查询攻击者与目标的距离，不在攻击范围则对寻路系统下达寻路移动指令，在攻击范围则下达攻击指令，切换时会命令停止移动或停止攻击。
技能系统在收到含有位移效果的技能时，不考虑寻路和碰撞，而是直接命令移动系统直接沿着技能所指的方向进行移动。
控制器系统为实体上的技能数组注册快捷键，输入触发对应技能的释放，包括移动、停止.

当自动攻击指令、移动指令、位移技能指令争抢最终移动系统的控制权时，比如自动攻击向目标靠近时收到了移动指令需要停止攻击改为移动，当位移技能位移时攻击指令和移动命令都会被忽略，，传统命令式的设计需要移动指令中手动停止自动攻击行为、移动命令需要检查是否处于位移状态。当越来越多的指令争抢移动控制权的时候，每个系统都需要知道其它系统的存在，那么该如何使用声明式的设计来解决这个问题呢
